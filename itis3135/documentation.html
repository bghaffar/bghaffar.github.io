Meeting the Specifications

This website meets the following specifications:

  • Interactive functionality using JavaScript:
    • 1. Contact Form Validation and Confirmation - This JavaScript checks the form's validity when the user submits it. If the form is valid, it hides the form and displays a confirmation message, simulating the form submission process.
    • 2. Interactive Image Toggling - Using jQuery, clicking on the buttons toggles the visibility of images such as the site map and wireframe, providing a dynamic user experience.
    • 3. AJAX Simulation and Footer Year Injection - The AJAX functionality simulates a response by updating the page content with a message, and the footer automatically updates the year using JavaScript to reflect the current year dynamically.
  • The site follows the CRAP principles in the following ways:
    • Contrast: The site uses contrasting colors like the dark header background (#1b2a49) against light text (#fff) for visibility. Links are styled to change color on hover, improving interactivity and highlighting user actions. Form fields have clear contrast between background and borders to ensure easy readability.
    • Repetition: Consistent design elements, like the navigation bar, footer, and form styling, are repeated throughout the site. These elements share the same colors, borders, and typography, creating a unified experience.
    • Alignment: Content is aligned consistently, especially in the navigation menu, forms, and footer. The text in the header and footer is centered, ensuring symmetry. Form fields and buttons are aligned vertically, improving usability.
    • Proximity: Related content, such as form labels, input fields, and submit buttons, are grouped together with clear spacing. This helps the user quickly identify related information. For example, in the contact form, labels and inputs are close together, making the form intuitive to fill out.

Feedback and Updates

Feedback received during critiques:

  • 1. Add a user feedback form to allow clients to tailor their workouts based on their personal needs.
  • 2. The gallery lacked sufficient images to effectively demonstrate different muscle groups and proper form.

How I updated my website based on this feedback:

  • 1. I added a user feedback form, allowing users to provide specific workout preferences. I implemented this using interactive JavaScript to enhance the user experience.
  • 2. I expanded the gallery by adding more images. I also included captions on the images so that when users hover over them, they receive instructions on how to perform the exercises correctly.

Meeting Stakeholder Needs and Future Plans

This website addresses the needs of my stakeholders by providing a clean, simple, and easy-to-use platform for beginners to create personalized workout plans. Faridun, who reviewed the site, praised its simplicity and effectiveness, noting that it allows users to quickly start their fitness journey and contact him for a tailored fitness plan.

Future plans for the website include:

  • Expanding the gallery to include more exercises and muscle groups for a more comprehensive workout demonstration.
  • Refining the user feedback form for even more detailed customization of workout plans.
  • Adding additional features to enhance user engagement, such as progress tracking and a community forum for sharing fitness tips.
